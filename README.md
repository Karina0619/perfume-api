Что внутри репозитория:
server.js - сам код на Node.js (с Express);
package.json и package-lock.json - это для установки библиотек;

Также есть скриншоты всех запросов: регистрация, логин, добавление товара, список товаров, заказ, отзывы
Были использованы методы: GET и POST.

Что работает:
1) Регистрация и вход по токену;
2) Админ может добавлять товары, обычный пользователь - нет;
3) Ограничение на запросы: максимум 100 в час;
4) Можно создать заказ и оставить отзыв.

В API были предусмотрены следующие коды ошибок:
400 - не заполнены обязательные поля;
401 - требуется авторизация (нет токена или он недействителен);
403 - доступ запрещён (недостаточно прав);
404 - ресурс не найден;
409 - пользователь с таким email уже существует;
429 - слишком много запросов (превышен лимит 100 в час).

Стратегия сопровождения API:
1) Версионирование (/api/v1/),каждая новая версия API будет иметь свой номер в адресе
2) Обратная совместимость - минимум 6 месяцев после выхода новой версии старая продолжает работать
3) Публичный Changelog (список изменений), все изменения будут записываться в отдельный файл
4) Уведомление разработчиков об изменениях по email
